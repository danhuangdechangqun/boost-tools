版本 日期 变更内容 作者

名词说明

名词 定义
Epic（史诗） 粒度较大的需求单元，通常对应一个模块或页面，需拆解为多个用户故事
User Story（用户故事） 粒度较小的需求单元，描述单一场景下用户的目标和系统应做什么
EARS格式 Easy Approach to Requirements Syntax，一种规范化的需求描述语法
验收标准 用于检验用户故事是否完成的检查项列表
依赖关系 表示一个用户故事必须在另一个用户故事完成后才能开始
P0 必须完成，没有它系统无法上线或核心流程阻塞
P1 重要功能，第一个版本建议包含，缺失会影响用户体验
P2 优化性功能，可在后续迭代补充，缺失不影响核心流程
P3 增强性功能，根据资源情况决定是否实现，可长期搁置
Aviator Google开源的表达式引擎，支持运算符、函数、变量等表达式语法
Cron Linux系统的定时任务表达式格式，由秒、分、时、日、月、周六个字段组成
LLM Large Language Model，大语言模型，如GPT、Claude、文心一言等
holiday-cn 中国法定节假日数据JSON库，包含每年放假/补班信息

二、需求背景

2.1 现状问题

问题类型 具体描述 影响程度
工具分散 JSON格式化、正则生成、UUID生成等工具分布在不同网站，需要频繁切换浏览器标签 高
周报效率低 每周手动整理已完成任务，耗时且容易遗漏，缺乏自动化生成机制 高
文档阅读耗时 长文档（Word/PDF）阅读耗时，难以快速定位重点内容 中
表达式生成困难 Aviator、Cron、正则等表达式语法复杂，容易出错，需要反复查阅文档 中
账号密码管理 多个账号密码分散记录在备忘录、文档中，查找不便 中
提示词重复编写 常用的LLM提示词每次都需要重新编写或复制粘贴，缺乏模板化管理 低

2.2 为什么现在做

  • 用户需求明确：开发者/办公人员日常工作场景痛点清晰，一站式解决方案需求强烈
  • 技术可行：Electron技术成熟，LLM API生态完善，本地存储方案稳定
  • 节假日数据现成：holiday-cn子模块提供完整的中国节假日数据，无需额外开发
  • 轻量化趋势：用户期望"不要太臃肿、快速唤起"的轻量工具，符合当前产品设计趋势

三、需求目标

目标类型 描述 衡量指标 目标值
效率提升 整合15个常用工具，减少工具切换时间 平均操作耗时 单次操作≤3秒
周报自动化 基于TodoList自动生成周报，识别节假日智能调整发送时间 周报生成时间 手动生成≤5秒
快速唤起 系统托盘常驻+快捷键唤起，不打断用户工作流 唤起响应时间 ≤500ms
本地化存储 所有数据本地存储，用户可控同步GitHub 数据安全性 本地存储100%可控
LLM能力集成 支持多格式LLM API（OpenAI、Claude、文心一言等） API兼容数 ≥3种主流格式

  • 数据更新：实时更新任务的group字段

📌 涉及角色：所有用户 📋 前置条件：TodoList中存在任务

详细规则描述

  • 触发条件：用户双击任意任务卡片（包括已完成任务）
  • 显示内容：任务标题、任务描述

功能：未完成待办（昨日遗留）

📌 涉及角色：所有用户 📋 前置条件：昨日有未完成任务

详细规则描述

  • 自动迁移：每天0点，系统将昨日"今日待办"中未完成的任务移动到"未完成待办"区域
  • 自动降级：每日0点，"明日待办"中的任务自动移动到"今日待办"
  • 视觉区分：未完成待办使用黄色/琥珀色背景，区别于普通待办
  • 拖拽支持：未完成待办可拖拽到今日/明日/下周分组，拖拽后移除黄色样式

功能：周报自动生成（BT-005）

📌 涉及角色：所有用户 📋 前置条件：已配置LLM API，本周有已完成任务

详细规则描述

  • 默认时间：每周五17:00
  • 节假日判断：读取holiday-cn子模块JSON数据
  • 调整规则：
    • 若周五放假：周四17:00触发
    • 若周六补班：周六17:00触发
  • 周报内容：汇总周一至触发时间的已完成任务，按日期分组
  • 输出格式：Markdown或纯文本（用户可选）

九、异常与边界处理

异常场景 触发条件 系统响应（EARS格式） 用户提示
LLM调用失败 API地址错误、Key无效、网络异常 当LLM API调用返回错误时，系统应捕获错误信息并显示错误提示 "大模型调用失败，请检查API配置或网络连接"
LLM调用超时 请求超过30秒未响应 若LLM API调用超过30秒未响应，系统应终止请求并提示超时 "请求超时，请稍后重试" + 重试按钮
文件上传超限 文件大小超过50MB 当用户上传文件大小超过50MB时，系统应拒绝上传并提示限制 "文件大小超过限制（50MB），请选择较小的文件"
文件格式错误 上传非Word/PDF文件 当用户上传非Word/PDF格式文件时，系统应拒绝上传并提示格式要求 "仅支持Word/PDF格式，请选择正确格式文件"
JSON/XML格式错误 输入文本不符合JSON/XML语法 当用户输入的JSON/XML文本存在语法错误时，系统应解析错误位置并提示 "第X行：缺少逗号" 或具体错误信息
周报无任务 触发周报生成时无已完成任务 当周报生成触发时本周无已完成任务，系统应提示无法生成 "本周暂无已完成任务，无法生成周报"
节假日数据缺失 holiday-cn子模块数据缺失或未更新 若节假日数据文件不存在或年份缺失，系统应按默认周五发送并提示 "节假日数据未更新，请检查holiday-cn子模块"
搜索无结果 笔记/账号密码搜索无匹配 当用户搜索笔记或账号密码无匹配结果时，系统应显示空状态提示 "暂无匹配结果"
快捷键冲突 用户设置的快捷键已被其他应用占用 当用户设置的快捷键注册失败时，系统应提示冲突并建议更换 "快捷键已被其他应用占用，请更换"
数据文件损坏 JSON数据文件格式损坏无法读取 当数据文件读取失败时，系统应提示用户并提供恢复选项 "数据文件读取失败，是否恢复默认数据？" + 恢复按钮

十、失败时的回滚方案

失败场景 回滚方案
应用启动失败 检查依赖完整性，提供日志输出，用户可查看启动错误信息
数据保存失败 保留操作缓存，下次启动时自动恢复未保存数据
LLM配置无效 保存配置前进行连接测试，测试失败不允许保存，保留上一版有效配置
版本升级失败 提供手动下载旧版本链接，用户可回退到稳定版本
